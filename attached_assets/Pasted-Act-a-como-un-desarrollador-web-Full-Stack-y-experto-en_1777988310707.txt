Actúa como un desarrollador web Full-Stack y experto en UI/UX. Necesito que desarrolles el código (HTML, CSS y JavaScript, o el framework que consideres óptimo para integrarse en una web moderna) para un minijuego llamado "Tinder de Propuestas".

El diseño debe ser mobile-first (pero responsive para PC), moderno, dinámico y fluido, utilizando animaciones suaves para deslizar tarjetas.

Estructura y Flujo del Juego:

1. Pantalla de Inicio (Intro):

Debe mostrar unas instrucciones breves: "Estas propuestas son para un consejo de estudiantes, de cara a las elecciones del 14 de mayo. Recuerda que lo que haga un consejo puede realmente incidir en la vida de un estudiante."

Un botón grande y llamativo que diga: "Empezar".

2. Pantalla de Juego (Mecánica Tinder):

Al empezar, el sistema debe seleccionar 15 propuestas al azar de una base de datos que te proporcionaré más abajo (mezclando "buenas" y "malas" sin indicar de qué tipo son).

Las propuestas aparecerán en formato de tarjeta (Card).

Interacción: El usuario debe deslizar (swipe) la tarjeta a la derecha si le gusta la propuesta, o a la izquierda si no le gusta. En PC, se puede hacer con el ratón o con botones inferiores de (❌ / 💚).

El sistema debe llevar el recuento de cuántas de las propuestas que el usuario ha deslizado a la derecha (Me gusta) pertenecen a la categoría de "Propuestas Buenas (Canvi)".

3. Pantalla de Resultados (Outro):

Al terminar las 15 tarjetas, debe aparecer una pantalla de resultados.

Mensaje 1: "¡Has hecho match! El X% de las propuestas que te han gustado son propuestas de canvi." (Donde X es el porcentaje calculado).

Tras unos segundos o mediante una animación fluida, el mensaje debe cambiar o revelar un Call to Action (CTA): "Recuerda votar a canvi el 14 de mayo para mejorar la UJI."

Un botón para "Volver a jugar".
